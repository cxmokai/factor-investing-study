# 讲义编写指南

> 本文档记录 Day 1 讲义的编写思路，用于复现后续 Day 2-11 的讲义。

---

## 一、编写流程

```
第1步：确定当天内容范围
  └→ 查看「金融知识学习计划.md」中该天的模块、知识点、对应书籍章节

第2步：列出所有知识点
  └→ 从对应的书籍章节中提取核心概念（参考「资料/书籍目录.md」）
  └→ 只保留因子投资必需的，跳过无关内容

第3步：排列知识点顺序
  └→ 原则：简单→复杂，具体→抽象，感性→理性
  └→ 例：Day 1 先讲"股票是什么"（具体），再讲"定价理论"（抽象）

第4步：为每个知识点编写内容（见下方"单个知识点的写法"）

第5步：编写自测题（覆盖所有关键知识点）

第6步：检查完整性
  └→ 读完讲义后，是否还需要翻书？如果需要，说明内容不够完整
  └→ 所有 Mermaid 图是否语法正确
```

## 三、单个知识点的写法

每个知识点按以下顺序组织，**不是每一步都必须**，根据概念复杂度取舍：

### 1. 一句话定义（必须）

用加粗的一句话给出最简定义，让人一秒抓住核心。

> **市值 = 股价 × 总股数**

### 2. 为什么要学这个（必须）

和因子投资的联系。如果和因子投资没有直接关系，说明"了解即可"。

> 这个概念极其重要——"市值因子"是因子投资最经典的因子之一。

### 3. 直觉类比（复杂概念必须）

用生活中的例子让人秒懂，避免纯抽象描述。

> 你去菜市场买苹果。苹果的"内在价值"取决于品种、新鲜度、口感……

好的类比原则：
- 来自日常生活，不需要额外知识
- 和原概念在结构上对应
- 简短，3 句话以内说完

### 4. Mermaid 图（关系/流程/对比时必须）

使用场景：

| 场景 | 图类型 | 例子 |
|------|--------|------|
| 概念分类 | `graph TD`（树形） | 金融市场 → 货币/资本/衍生品 |
| 流程/因果 | `graph TD` 或 `graph LR` | 坚实基础理论 → 分析基本面 → 判断低估 → 买入 |
| 两者对比 | `graph LR` + subgraph | 坚实基础理论 vs 空中楼阁理论 |
| 线性光谱 | `graph LR`（左右排列） | 被动投资 ← 因子投资 → 主动投资 |

Mermaid 书写注意：
- 节点文本用双引号包裹，换行用 `<br>`
- 不要使用 emoji
- subgraph 名称不要用中文引号
- 保持简洁，单图节点不超过 10 个

### 5. 具体数字/例子（尽量有）

用真实公司、真实数据，让概念落地。

> 贵州茅台：股价约 1500 元 × 总股本约 12.6 亿股 ≈ 市值 1.9 万亿元

注意：数字会过时，用"约"字，不需要精确到个位。

### 6. 表格对比（有对比关系时必须）

两个以上概念做对比时，必须用表格。

| 维度 | 概念A | 概念B |
|------|-------|-------|
| 维度1 | ... | ... |
| 维度2 | ... | ... |

### 7. 和因子投资的映射（重要概念必须）

明确告诉读者"这个概念在第 N 周会怎么用"。

> **因子投资中的"市值因子"（Size Factor）**：学术研究发现，长期来看小盘股的平均收益高于大盘股。
> 我们在第 7 周会详细研究。

## 四、自测题的写法

### 格式

```
自测 N：问题？
→ 你的答案：_______________
→ 正确：简短的标准答案。
```

### 原则

- 每个核心知识点至少对应一道自测题
- 答案必须简短（1-2 句话），不要写长段
- 题目数量 = 学习时间（小时）× 5，即 2 小时对应约 10 道题
- 末尾给评分标准：90% 以上 = 完美，70-80% = 回看错的部分，60% 以下 = 需要复习

## 五、写作原则

### 必须做的

1. **完全自包含**：读完讲义不需要翻任何书。所有知识点直接写在文档里
2. **每个概念都连接到因子投资**：读者要始终知道"我为什么在学这个"
3. **先具体后抽象**：先给例子/类比，再给定义/公式
4. **预判读者疑问**：金融小白会在哪里卡住？提前解释
   - Day 1 实际遇到的问题：股息vs分红区别、优先股vs普通股、A股名称由来、蓝筹含义
   - 这些"小白问题"应该在讲义中主动覆盖
5. **标记深度边界**：明确告诉读者"这个概念今天只需了解到这个程度"

### 不能做的

1. **不写"请搜索 XXX"** — 要搜什么，直接把内容写出来
2. **不写"请翻阅《XX》第 N 章"** — 需要的知识直接写在讲义里
3. **不堆术语** — 每个术语第一次出现时必须解释
4. **不讲因子投资不需要的内容** — 比如 Day 1 跳过了衍生品细节、货币市场细节
5. **不用 emoji**

## 七、预判"小白问题"清单

Day 1 编写过程中，用户实际提出了以下问题。后续讲义应主动覆盖类似的"想当然但其实不懂"的问题：

| 用户问题 | 说明 |
|---------|------|
| 股息和分红有什么区别？ | 术语混用造成的困惑，讲义应主动区分 |
| 买普通股能拿到分红吗？ | 知道"有分红"但不确定自己能不能拿到 |
| 券商软件买的是普通股还是优先股？ | 担心买错 |
| A 股为什么叫 A 股？ | 对名称本身的好奇 |
| 蓝筹股的"蓝筹"是什么意思？ | 对术语来源的好奇 |

**规律：** 小白的问题往往不是"这个理论怎么推导"，而是：
1. **两个相似术语的区别**（股息 vs 分红、普通股 vs 优先股）
2. **这和我有什么关系**（我能拿到分红吗、我买的是哪种）
3. **名字/术语的来源**（为什么叫 A 股、蓝筹是什么意思）

写讲义时应主动在概念旁边回答这类问题。
